Организация проекта

  1. Исходники лежат в папке verstka/src/

    • data/ (json, где лежит контент).
    • images/ (картинки), тестовые картинки лежат в папке temp, их можно удалять, иконки лежат в папке svg/, из них формируется свг-спрайт.
    • js/ делится на libs/, где лежат сторонние библиотеки (slick, sly и т.д) и файл dsklad-init.js, где подключаются эти файлы.
    • pug/ В корне лежат страницы - index.pug, basket.pug.
      Cлужебные шаблоны лежат в папке templates, это основной шаблон layout.pug, инклюды - incl-header.pug, incl-footer.pug и т.д.
    • папка pug/kit/ - страницы для кита
    • папка sass/ состоит из main-styles.sass, куда подключаются разбитые по блокам стили. В templates/ хранятся переменные с цветами, размерами - _variables.sass, _colors.sass, _mixin.sass, _grid-system.sass, _sprite_template.sass


  2. Устройство gulpfile.js, таски
    • pug - запускает сборку html, данные тянутся из data/data.json, либо лежат в файлах pug
    • style - сборка стилей, подключен автопрефиксер, стили с медиа запросами помещаются в конец файла
    • js - перекладывание js в папку build
    • svg - формирует спрайт и стили для спрайта. Иконки должны лежать в папке images/svg, стили подключаются с помощью классов .icon-svg.ic-имя_иконки. В стилях иконка располагается в :before
    • img - оптимизация картинок
    • clean - очистка папки build
